Overview

프로젝트 개요

IT 공모전 정보가 여러 플랫폼에 분산되어 있어 매번 확인하기 비효율적인 문제를 해결합니다.

핵심 기능
수집 5개 플랫폼에서 공모전 데이터를 자동 크롤링
정제 중복 제거, 상태 관리, 카테고리 분류 자동 처리
알림 새 공모전 등록 시 Discord Webhook으로 알림
배포 GitHub Actions로 빌드부터 배포까지 완전 자동화

데이터 처리 파이프라인
1

병렬 수집

5개 스크래퍼 순차 실행, 실패한 소스는 기존 데이터 유지

2

중복 제거

ID 매칭 + 제목 정규화 비교 (특수문자 · 공백 제거 후 비교)

3

Status 갱신

deadline < today → completed, 아니면 active

4

Retention 정리

마감 후 90일 지난 항목 자동 삭제

5

정렬 및 저장

마감일순 정렬 → data/contests.json 출력

Discord Notification

Discord 알림 시스템

# crawler/notify.py # 1. 현재 active 공모전 로드 contests = load("data/contests.json") active = [c for c in contests if c.status == "active"] # 2. 이전 알림 ID와 비교 → 새 항목만 추출 notified = load("data/notified.json") new_contests = [c for c in active if c.id not in notified] # 3. Discord Embed 생성 및 Webhook POST embeds = [build_embed(c) for c in new_contests] send_webhook(url, embeds) # 10개씩 배치 # 4. 알림 완료 ID 저장 (중복 방지) save(notified | new_ids, "data/notified.json")
Embed 구성
제목 + URL 주최자 마감일 (D-Day) 카테고리 썸네일

DISCORD_WEBHOOK_URL은 GitHub Secrets에 저장. 환경변수 미설정 시 스킵하여 로컬에서도 안전하게 동작.

Frontend

프론트엔드 구성

Next.js 16 — App Router + Static Export

output: "export"로 빌드 시점에 모든 페이지를 HTML로 생성. 서버 런타임 없이 GitHub Pages에 배포.

데이터 흐름

data/contests.json → 빌드 시 import로 직접 로드 → generateStaticParams()로 각 공모전 상세 페이지 사전 생성. 별도 API 호출 없이 JSON을 직접 소비하는 구조.

Tailwind CSS 4

Utility-first 반응형 디자인. sm:grid-cols-2, lg:grid-cols-3로 디바이스별 그리드 대응.
